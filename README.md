# Bite-Size Python

Short demos that are ~10 min each.

---

### Building a Productive Repo

- [ ] **`uv` for dependency management:** one command to manage your project and reproduce your entire environment
- [ ] **`git` version control:** the git commands and workflows worth knowing
- [ ] **One formatter, zero style debates:** writing code you'll want to show others
- [ ] **Repo structure that enforces intent:** a layout that makes it easier to think clearly about your work
- [ ] **Building a reusable library:** turn a collection of scripts into something you can import, share, and depend on
- [ ] **Git submodules:** update your dev tools in one place, and every repo gets them

---

### Writing Better Functions

- [ ] **Keyword-only arguments:** never pass the wrong argument silently again
- [ ] **Thinking functional:** keep functions focused on one job and testing becomes trivial
- [ ] **Unit tests:** catch mistakes in your pipeline before they corrupt your results
- [ ] **Type validation:** stop bad inputs at the door; clear errors every time

---

### Modelling Your Data

- [x] **Structured returns:** named fields mean you never guess argument order, and you can catch typos at runtime
- [ ] **How to name things:** the right name is worth more than any comment
- [ ] **Type registries with nested classes:** keep related types grouped and discoverable, not scattered across flat alias files
- [ ] **Classes as operator pipelines:** a few special methods and your class behaves like it was always part of the language
